# 研究向补充：微调入口、数据衔接与评测汇总

本文档给 **想把教程里的脚手架升级成可写进报告/论文的实验** 的读者用，不替代正式论文写作课，只补 **工具链与自检清单**。

**前置**：请先读 [第十章第十一节「给有想法、想做成可发表工作的读者」](./chapter10/第十章%20学习路线与开源项目实战建议.md)（基线、消融、复现边界）。

## 一、本仓库 JSONL 与「训练框架」怎么接

- [第四章](./chapter4/第四章%20数据、训练与微调.md) 与 `docs/chapter4/code/sample_multimodal_sft.jsonl` 展示的是 **对话式多模态指令** 的常见字段习惯（`messages` + 图/文 `content`）。
- 公开微调框架（如下）通常还有自己的 **manifest / 注册表 / 数据集类**，往往需要你写 **几十行转换脚本**：读本仓库 JSONL → 写出框架可读格式，或继承其 `Dataset` 按行解析。**不必**强求本教程内置完整训练栈；选一款维护活跃、文档清晰的框架跟到底更省事。

可参考（链接以仓库文档为准，版本迭代快）：

- [Hugging Face Transformers 文档](https://huggingface.co/docs/transformers)：官方示例与 `Trainer` / 多模态模型类名。
- [LLaMA-Factory](https://github.com/hiyouga/LLaMA-Factory)：社区常用的统一微调入口，常覆盖多种 VLM 配方；对照其 **数据集格式说明** 做转换。
- [ms-swift](https://github.com/modelscope/ms-swift)：ModelScope 侧的多模态微调与部署工具链，同样需对照其数据格式。

**实践建议**：先在你选定的框架里跑通 **官方最小 demo**，再把 [第四章](./chapter4/第四章%20数据、训练与微调.md) 的样本 **逐条对照** 字段是否被正确读入（batch 里图像路径、对话轮次、mask 等）。

## 二、离线评测结果如何汇总成表

[第五章](./chapter5/第五章%20评测体系与工程选型.md) 中 `eval_vlm_dataset.py` 会写出 `eval_results.jsonl`。仓库提供标准库脚本：

```bash
# 单文件：错误率、粗粒度 reference 子串命中、耗时、按 tags 分桶
python docs/chapter5/code/summarize_eval_results.py docs/chapter5/code/eval_results.jsonl

# 未跑通 API 时，可用仓库内示例输出试脚本（字段与真实 eval 一致）
python docs/chapter5/code/summarize_eval_results.py docs/chapter5/code/sample_eval_results_output.jsonl

# A/B：例如两模型各跑一遍，输出可粘贴进报告的 Markdown 表
python docs/chapter5/code/summarize_eval_results.py run_a.jsonl --compare run_b.jsonl --label 模型A --label 模型B --markdown
```

**Windows 终端**：脚本会尝试把标准输出设为 UTF-8；若仍乱码，可先执行 `chcp 65001`、使用 PowerShell 7+，或用 **`--write report.md`** 把全文（与终端一致）写成 UTF-8 文件再用编辑器打开。

**说明**：「粗粒度参考命中率」与第五章正文一致——仅适合开放问答的**粗筛**；正式论文中请写明是否另有人工裁定或 BLEU/ROUGE 等（若领域常用），并说明局限。

## 三、实验记录最小清单（可贴进 `REPRODUCE.md`）

1. **数据**：评测 JSONL 版本、条数、`tags` 定义；训练集若与教程格式不同，附转换脚本提交或链接。  
2. **模型**：checkpoint 名或路径、发布方、是否量化、推理 `max_tokens`。  
3. **环境**：Python 版本、关键包版本（`pip freeze` 选摘即可）、CUDA / 驱动概要。  
4. **命令**：从原始数据到 `eval_results.jsonl` 的完整命令行（含 `OPENAI_BASE_URL` 或本地推理入口）。  
5. **随机性**：若有训练，记录 seed；若仅推理，记录是否固定采样参数。
