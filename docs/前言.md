# 前言

## 一、为什么要系统学习多模态大模型

大语言模型解决的是“基于文本进行理解和生成”的问题，而多模态大模型（MLLM, Multimodal Large Language Model）试图进一步处理图片、文档、截图、表格、视频帧、语音等多种输入形式。它的真正价值，不是“多接了一个图片入口”，而是让模型逐步具备接近真实世界任务的感知能力。

现实应用里，用户很少只提供纯文本。更常见的输入是：

- 一张报错截图，外加一句“这是什么意思”
- 一份发票或合同，外加一句“帮我提取关键信息”
- 一张商品图，外加一句“帮我写标题和卖点”
- 一段视频封面、多张图片、语音说明、结构化表格的组合输入

如果你只会文本模型，你能做的事情会被输入形式限制；如果你理解了多模态模型，你就能开始做真正贴近业务的 AI 系统。

## 二、本项目适合谁

这份教程主要面向三类读者：

1. 已经学过 LLM，希望进一步理解 VLM/MLLM 的开发者。
2. 做应用或 Agent，希望把图像、文档、截图能力接入系统的工程师。
3. 想做教程、开源项目、知识库整理的学习者与学生。

## 三、阅读前建议具备的知识

为了更顺畅地学习，建议你具备以下基础：

- Python 基础语法
- `PyTorch` 或 `Transformers` 的基本使用经验
- 对 Transformer、Token、Embedding、SFT 这些概念有初步认知
- 知道 LLM 的基本调用方式，例如本地推理、OpenAI 兼容接口或 API 网关

如果你暂时还不熟悉这些内容，也可以先按“概念优先、代码其次”的方式阅读，再逐步补齐。

## 四、如何使用这份教程

建议用下面这套方法学习：

### 1. 先建地图

先读第一章到第三章，搞清楚：

- 模态是什么
- 为什么图像不能直接喂给 LLM
- 视觉编码器、投影层、语言模型分别负责什么

### 2. 再看数据和训练

第四章会帮助你理解：

- 为什么多模态训练很依赖数据配方

第七章和第八章提供了最小可运行示例。你不一定要先把所有理论读完再动手，很多概念是在实践过程中才真正变得具体。

## 五、学习时要避免的几个误区

### 误区 1：把“能收图片”当成“理解图像”

很多多模态模型的能力边界并不来自“有没有图片输入”，而来自：

- 视觉编码器是否足够强
- 对齐是否足够好
- 数据分布里是否包含你关心的任务
- 推理链路里是否保留了必要的细粒度信息

### 误区 2：只看排行榜，不看任务结构

一个模型在综合榜单上更高分，不代表它一定适合你的业务。文档理解、OCR 密集场景、中文截图、图表推理、多图对话，都可能需要不同的模型偏好。

### 误区 3：只会调 API，不理解系统结构

如果你只停留在“上传图片然后拿答案”，很难真的判断问题出在：

- Prompt
- 数据
- 模型架构
- 推理配置
- 部署服务

理解结构，才有能力真正定位问题。
