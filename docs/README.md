# Start-MLLM

欢迎来到 `Start-MLLM`。

这是一份面向中文学习者的多模态大模型教程，主线是“先搭框架，再做实践”。整套内容更像一条从 `LLM 入门视角` 走向 `MLLM 工程视角` 的路线图。

这一套内容不是照模板堆出来的，每章重点都不一样：有的偏概念拆解，有的偏工程判断，有的偏动手实践。按你当前阶段挑读即可，不必拘泥固定顺序。

## 这份教程解决什么问题

很多人学多模态时会卡在三个位置：

1. 论文名词很多，但不知道每个模块在系统里扮演什么角色。
2. 知道模型能“看图说话”，但不知道数据、训练、部署分别怎么落地。
3. 会调用 API，但不会把多模态能力接入自己的应用或 Agent。

本项目就是围绕这三个问题展开。

阅读方式可以参考：

- 先读主线，不要一开始深陷细节。
- 每学完一章，立即做“章末练习”中的 1 到 2 个任务。
- 第四章、第五章、第七章、第八章、第九章都配了更偏实战的脚本或脚手架，适合作为你自己项目的起点。

## 配套脚手架

如果你更偏实践，先看这些现成脚手架会更顺手：

- 第四章：`create_placeholder_images.py` + 最小多模态 SFT JSONL + `prepare_dataset_manifest.py`（先跑通路径再换真图）
- 第五章：`create_placeholder_images.py` + `eval_vlm_dataset.py`（支持 `-h` / `--dataset` 等，未装 `openai` 时也可先看帮助）
- 第七章：本地 `Transformers` 与 OpenAI 兼容客户端；图片路径由 `docs/learner_paths.py` 统一解析（可不设 `IMAGE_PATH` 时用第五章占位图，也便于在仓库根目录运行）
- 第八章：多模式 `Gradio` Demo
- 第九章：最小多模态 Agent 骨架

更细的「第一周怎么迭代」见 [前言](./前言.md) 里「学习者视角：第一周可以怎么迭代」一节。
